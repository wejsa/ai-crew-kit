# 구독 커머스

## 개요

상품을 일회성이 아닌 정기적으로 배송하는 구독형 커머스 모델입니다. 구독 주기 관리, 결제 재시도, 일시정지/재개가 핵심입니다.

## 구독 상태 머신

```
created → active → paused → active (재개)
              ↘ past_due → active (결제 성공)
              ↘ past_due → canceled (결제 실패 N회)
              ↘ canceled → expired (유예 기간 종료)
```

| 상태 | 설명 | 전이 조건 |
|------|------|----------|
| created | 구독 생성 | 최초 결제 전 |
| active | 정상 구독 중 | 결제 성공 |
| past_due | 결제 실패 (유예 중) | 정기 결제 실패 |
| paused | 일시 정지 | 사용자 요청 |
| canceled | 해지 예약 | 사용자 해지 또는 결제 실패 한도 초과 |
| expired | 만료 | 해지 후 유예 기간 종료, 데이터 보존만 |

### 허용 전이

| From | To | 조건 |
|------|----|------|
| created | active | 최초 결제 성공 |
| active | paused | 사용자 일시정지 요청 |
| active | past_due | 정기 결제 실패 |
| active | canceled | 사용자 해지 요청 |
| paused | active | 사용자 재개 요청 |
| past_due | active | 재시도 결제 성공 |
| past_due | canceled | 재시도 3회 실패 |
| canceled | expired | 유예 기간(30일) 종료 |

## 결제 주기

| 주기 | 설명 |
|------|------|
| 주간 | 매주 동일 요일 |
| 격주 | 2주 간격 |
| 월간 | 매월 동일 일자 (말일 보정: 31일→28/29/30) |
| 분기 | 3개월 간격 |

### 결제 실패 재시도

| 재시도 | 시점 | 행동 |
|--------|------|------|
| 1차 | 실패 후 1일 | 자동 재시도 |
| 2차 | 실패 후 3일 | 자동 재시도 + 이메일 알림 |
| 3차 | 실패 후 7일 | 자동 재시도 + SMS 알림 |
| 실패 확정 | 3차 실패 | canceled 전이, 배송 중단 |

## 정기 배송 관리

| 규칙 | 설명 |
|------|------|
| 배송 예정일 산출 | 결제 성공일 + 배송 리드타임 |
| 건너뛰기 | 다음 1회 배송 건너뛰기 (구독 유지) |
| 수량 변경 | 다음 배송부터 수량 변경 적용 |
| 상품 변경 | 다음 배송부터 상품 교체 (동일 카테고리 내) |
| 배송지 변경 | 배송 확정 전까지 변경 가능 |

## 프로레이션 (일할 계산)

| 상황 | 처리 |
|------|------|
| 월 중 플랜 업그레이드 | 남은 일수 비례 차액 청구 |
| 월 중 플랜 다운그레이드 | 다음 결제 주기부터 적용 (당월 환불 없음) |
| 월 중 해지 | 당월 잔여 기간 이용 가능, 환불 없음 |

- 프로레이션 계산은 **BigDecimal** 필수
- 일할 계산: `(월 요금 / 해당 월 일수) × 잔여 일수`, HALF_UP 반올림

## 참고사항

- 구독 상태 변경은 반드시 이력 테이블에 기록 (변경 전/후 + 사유)
- 일시정지 최대 기간: 연속 90일 (초과 시 자동 해지)
- 해지 유예 기간(30일) 내 재가입 시 기존 데이터 복원
- 정기 결제는 PG사 빌링키 기반, 빌링키 만료 시 재등록 유도
