# 사용량 미터링 / 피처 게이팅

## 개요

플랜별 사용량 측정, 쿼터 관리, 초과 과금, 기능 게이팅을 정의합니다.

## 미터링 대상

| 대상 | 측정 단위 | 집계 주기 |
|------|----------|----------|
| API 호출 | 건수 | 분/시/일/월 |
| 스토리지 | GB | 일 (최대값) |
| 시트 (사용자 수) | 명 | 실시간 |
| 대역폭 | GB | 월 |
| 컴퓨팅 | 분 | 월 |

## 쿼터 / 리밋

### 플랜별 쿼터 테이블 (예시)

| 리소스 | Free | Starter | Pro | Enterprise |
|--------|------|---------|-----|-----------|
| API 호출/월 | 1,000 | 50,000 | 500,000 | 무제한 |
| 스토리지 | 1GB | 10GB | 100GB | 커스텀 |
| 시트 | 3명 | 10명 | 50명 | 무제한 |
| 웹훅 | 1개 | 5개 | 20개 | 무제한 |

### 리밋 유형

| 유형 | 동작 | 사용자 경험 |
|------|------|-----------|
| 소프트 리밋 | 경고 + 계속 허용 | 대시보드 경고, 이메일 알림 |
| 하드 리밋 | 요청 차단 (HTTP 429) | 에러 메시지 + 업그레이드 안내 |

### 리밋 알림 규칙

| 시점 | 알림 |
|------|------|
| 80% 도달 | 대시보드 경고 배너 |
| 90% 도달 | 이메일 알림 |
| 100% 도달 | 소프트: 이메일 + 인앱 / 하드: 요청 차단 |
| 120% 도달 (소프트) | 초과 과금 시작 알림 |

## 초과 과금

| 규칙 | 설명 |
|------|------|
| 적용 조건 | 소프트 리밋 플랜에서 쿼터 초과 시 |
| 단가 | 리소스별 초과 단가 (플랜에 정의) |
| 계산 | 초과 사용량 × 초과 단가 (BigDecimal, HALF_UP) |
| 청구 | 다음 인보이스에 초과 요금 항목 추가 |
| 상한 | 초과 요금 상한 설정 가능 (예: 기본 요금의 200%) |

## 피처 게이팅

### 피처 플래그 모델

| 구분 | 설명 |
|------|------|
| 플랜 게이팅 | 특정 기능이 특정 플랜 이상에서만 활성화 |
| 쿼터 게이팅 | 쿼터 소진 시 기능 비활성화 |
| 베타 게이팅 | 특정 테넌트에만 얼리 액세스 제공 |

### 피처 플랜 매트릭스 (예시)

| 피처 | Free | Starter | Pro | Enterprise |
|------|------|---------|-----|-----------|
| 기본 CRUD | ✅ | ✅ | ✅ | ✅ |
| API 접근 | ❌ | ✅ | ✅ | ✅ |
| SSO | ❌ | ❌ | ✅ | ✅ |
| 감사 로그 조회 | ❌ | ❌ | ✅ | ✅ |
| 커스텀 도메인 | ❌ | ❌ | ❌ | ✅ |
| SLA 보장 | ❌ | ❌ | ❌ | ✅ |

### 게이팅 검증 규칙

| 규칙 | 설명 |
|------|------|
| API 레벨 검증 | 요청 시 테넌트 플랜 확인 → 허용/차단 |
| UI 레벨 표시 | 비활성 기능은 잠금 아이콘 + 업그레이드 CTA |
| 다운그레이드 시 | 기존 사용 중이던 기능은 읽기 전용 전환 (삭제하지 않음) |

## 사용량 리포팅

### 테넌트 대시보드 항목

| 항목 | 설명 |
|------|------|
| 현재 사용량 / 쿼터 | 리소스별 사용률 (%) |
| 일별/월별 추세 | 시계열 그래프 |
| 예상 초과 알림 | 현재 추세 기반 쿼터 초과 예상일 |
| 인보이스 미리보기 | 현재까지 누적 요금 |

## 참고사항

- 미터링 데이터는 최소 1년 보존 (과금 분쟁 대비)
- 사용량 집계는 near-realtime (최대 5분 지연 허용)
- 피처 플래그 변경은 즉시 반영 (캐시 TTL 최대 60초)
- 쿼터 초과로 인한 서비스 중단 시 사전 알림 필수
