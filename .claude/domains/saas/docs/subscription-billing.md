# 구독 결제

## 개요

SaaS 플랫폼의 구독 과금 모델, 상태 관리, 플랜 변경, 인보이스를 정의합니다. 물리적 상품 정기 배송(ecommerce 구독 커머스)과 구분되는 서비스 접근 기반 구독입니다.

## 과금 모델

| 모델 | 설명 | 계산 |
|------|------|------|
| Flat-rate | 고정 월/연 요금 | 플랜 가격 × 1 |
| Seat-based | 사용자 수 기반 | 시트 단가 × 시트 수 |
| Usage-based | 사용량 기반 | 사용량 × 단가 |
| Tiered | 구간별 차등 단가 | Σ(구간별 사용량 × 구간 단가) |
| Hybrid | Flat base + 초과 사용량 | 기본 요금 + (초과분 × 단가) |

- 모든 금액은 **BigDecimal** 필수
- 세금 계산: 공급가액 + 부가세(10%) = 총액
- 외화 결제: 통화 코드(ISO 4217) 명시, 환율 고정 시점 기록

## 구독 상태 머신

```
trialing → active ⇄ past_due → canceled → terminated
  ↓          ⇅                     ↑
  canceled   paused ───────────────┘ (90일 초과)
```

| 상태 | 설명 | 서비스 접근 |
|------|------|-----------|
| trialing | 무료 체험 기간 | 전체 또는 제한 접근 |
| active | 정상 구독 | 전체 접근 |
| past_due | 결제 실패 (유예) | 전체 접근 (읽기만 또는 전체) |
| paused | 일시 정지 | 읽기 전용 |
| canceled | 해지 예약 (유예 기간) | 현재 주기 끝까지 접근 |
| terminated | 완전 해지 | 접근 차단 |

### 허용 전이

| From | To | 조건 |
|------|----|------|
| trialing | active | 결제 정보 등록 + 첫 결제 성공 |
| trialing | canceled | 사용자 취소 또는 트라이얼 기간 만료 |
| active | past_due | 정기 결제 실패 |
| active | paused | 사용자 일시정지 요청 |
| active | canceled | 사용자 해지 요청 |
| past_due | active | 재시도 결제 성공 |
| past_due | canceled | 재시도 3회 실패 (총 4회 시도) |
| paused | active | 사용자 재개 요청 + 결제 성공 |
| paused | canceled | 일시정지 90일 초과 |
| canceled | terminated | 유예 기간 종료 (현재 결제 주기 끝) |

## 결제 주기

| 주기 | 설명 | 할인 |
|------|------|------|
| 월간 | 매월 가입일 기준 | 기본 |
| 연간 | 매년 가입일 기준 | 월 대비 할인 (통상 ~20%) |

### 결제 실패 재시도

| 시도 | 시점 | 행동 |
|------|------|------|
| 최초 | 결제 예정일 | 정기 결제 → 실패 시 past_due |
| 재시도 1차 | +1일 | 자동 재시도 |
| 재시도 2차 | +3일 | 자동 재시도 + 이메일 알림 |
| 재시도 3차 | +7일 | 자동 재시도 + 이메일+인앱 알림 |
| 실패 확정 | 3차 실패 | canceled 전이 (총 4회 시도) |

## 플랜 변경

| 변경 유형 | 적용 시점 | 결제 처리 |
|----------|----------|----------|
| 업그레이드 | 즉시 | 프로레이션 차액 즉시 청구 |
| 다운그레이드 | 다음 결제 주기 | 당월 환불 없음 |
| 시트 추가 | 즉시 | 프로레이션 차액 즉시 청구 |
| 시트 감소 | 다음 결제 주기 | 당월 환불 없음 |
| 월→연 전환 | 즉시 | 연간 요금 청구 (잔여 월 프로레이션 크레딧) |

### 프로레이션 계산

```
프로레이션 = (신규 플랜 일 단가 - 기존 플랜 일 단가) × 잔여 일수

일 단가 = 월 요금 / 해당 월 일수 (BigDecimal, HALF_UP)
```

- 프로레이션 결과가 음수면 크레딧으로 적립 (환불 아님)
- 프로레이션 최소 금액: 100원 미만은 절사

## 인보이스

### 인보이스 항목 구조

| 항목 | 설명 |
|------|------|
| 기본 요금 | 플랜 기본 가격 |
| 시트 요금 | 추가 시트 × 단가 |
| 사용량 초과 | 초과분 × 단가 |
| 프로레이션 | 플랜 변경 차액 |
| 크레딧 적용 | 기존 크레딧 차감 |
| 세금 | 부가세 10% |
| **총액** | 공급가액 + 세금 - 크레딧 |

### 인보이스 상태

| 상태 | 설명 |
|------|------|
| draft | 생성, 미확정 |
| finalized | 확정, 결제 대기 |
| paid | 결제 완료 |
| void | 취소 (발행 오류) |
| refunded | 환불 |

## 참고사항

- 구독 상태 변경은 반드시 이력 테이블에 기록 (변경 전/후 + 사유 + 시각)
- 일시정지 최대 기간: 연속 90일 (초과 시 자동 canceled)
- canceled 유예 기간: 현재 결제 주기 종료일까지
- terminated 후 데이터 보존은 `data-lifecycle.md` 참조
- 테넌트 상태와의 관계: 구독 canceled → 유예 종료 → 테넌트 suspended (30일 후 terminated) → 데이터 보존 90일 후 완전 삭제 (data-lifecycle.md 참조)
