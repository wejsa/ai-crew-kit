# 온보딩 / 프로비저닝

## 개요

사용자 가입부터 테넌트 생성, 초기 설정, 트라이얼 시작까지의 온보딩 플로우를 정의합니다.

## 가입 플로우

```
회원가입 → 이메일 인증 → 테넌트 생성 → 프로비저닝 → 트라이얼 시작 → 대시보드 진입
```

### 가입 필수 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 이메일 | ✅ | 고유, 인증 필수 |
| 비밀번호 | ✅ | 최소 8자, 복잡도 규칙 |
| 조직명 | ✅ | 테넌트 표시명 |
| 리전 선택 | ✅ | 데이터 저장 리전 (변경 불가) |
| 약관 동의 | ✅ | 서비스 이용약관 + 개인정보 처리방침 |
| 플랜 선택 | ⚠️ | 트라이얼 시 생략 가능 |

## 프로비저닝 체크리스트

테넌트가 `provisioning → active` 전이하기 위해 완료해야 할 항목:

| 항목 | 설명 |
|------|------|
| 테넌트 레코드 생성 | tenants 테이블에 행 삽입 |
| DB 격리 설정 | RLS 정책 적용 또는 스키마/DB 생성 |
| 기본 역할 생성 | Tenant Admin, Manager, Member, Viewer 역할 자동 생성 |
| 가입자 Admin 할당 | 가입한 사용자를 Tenant Admin으로 설정 |
| 기본 설정 시딩 | 알림 설정, 타임존, 언어 등 기본값 |
| 웰컴 이메일 발송 | 가입 완료 + 시작 가이드 안내 |

### 프로비저닝 실패 처리

| 규칙 | 설명 |
|------|------|
| 롤백 | 생성된 모든 리소스 원복 (부분 프로비저닝 금지) |
| 재시도 | 최대 2회 자동 재시도 후 실패 확정 |
| 알림 | 실패 시 사용자에게 안내 + 운영팀 알림 |
| 상태 | 테넌트 terminated 전이 (실패 사유 기록) |

## 트라이얼 관리

| 규칙 | 설명 |
|------|------|
| 기본 기간 | 14일 (프로젝트별 설정 가능) |
| 기능 제한 | 플랜별 트라이얼 제한 정의 (시트 수, 스토리지, API 호출 등) |
| 연장 | 운영팀 수동 연장 (최대 1회, 14일) |
| 만료 처리 | 결제 정보 없으면 → 구독 canceled → 유예 후 구독 terminated → 테넌트 suspended |
| 전환 유도 | 만료 7일 전, 3일 전, 당일 알림 |

### 트라이얼 → 유료 전환

```
트라이얼 만료 전 결제 정보 등록
  → 첫 결제 성공 → 구독 trialing → active
  → 첫 결제 실패 → 구독 trialing → canceled
```

## 셀프서비스 설정

테넌트 Admin이 직접 관리하는 항목:

| 설정 | 설명 |
|------|------|
| 조직 프로필 | 조직명, 로고, 설명 |
| 커스텀 도메인 | 테넌트 전용 서브도메인 또는 커스텀 도메인 |
| 알림 설정 | 이메일/슬랙/웹훅 알림 채널 |
| 보안 설정 | 2FA 필수, IP 화이트리스트, 세션 타임아웃 |
| SSO 설정 | SAML/OIDC IdP 연동 (엔터프라이즈 플랜) |

## 참고사항

- 프로비저닝 소요 시간: RLS 방식 <3초, 스키마 방식 <30초, DB 방식 <2분
- 가입 시 수집하는 개인정보는 최소 수집 원칙 준수
- 트라이얼 기간 중 생성한 데이터는 유료 전환 시 유지
- SSO 설정은 엔터프라이즈 플랜 이상에서만 활성화
