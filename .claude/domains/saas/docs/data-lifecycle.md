# 데이터 생명주기

## 개요

테넌트 데이터의 생성, 보관, 내보내기, 삭제 전 과정을 관리합니다. GDPR 삭제권/이동권과 데이터 레지던시를 포함합니다.

## 데이터 분류

| 분류 | 예시 | 보존 기간 | 삭제 가능 |
|------|------|----------|----------|
| 개인정보 | 이름, 이메일, 연락처 | 서비스 이용 중 | ✅ (삭제권) |
| 서비스 데이터 | 프로젝트, 문서, 설정 | 테넌트 해지 후 90일 | ✅ (해지 시) |
| 결제 데이터 | 인보이스, 결제 이력 | 5년 (세법) | ❌ (법적 의무) |
| 감사 로그 | 접근/변경 이력 | 5년 (SOC2) | ❌ (법적 의무) |
| 분석 데이터 | 익명화된 사용 통계 | 무기한 | N/A (비식별) |

## GDPR 삭제권 (Right to Erasure)

### 삭제 요청 처리

```
사용자 삭제 요청 → 본인 확인 → 삭제 범위 산정
  → 법적 보존 의무 항목 제외 → 삭제 실행 → 폐기 증명 발급
```

| 규칙 | 설명 |
|------|------|
| 처리 기한 | 요청 접수 후 30일 이내 |
| 삭제 범위 | 개인정보 + 서비스 데이터 (법적 보존 제외) |
| 법적 예외 | 결제 데이터(세법 5년), 감사 로그(SOC2), 분쟁 증거 |
| 백업 포함 | 백업 데이터에서도 삭제 (다음 백업 주기에 자동 제거) |
| 제3자 통지 | 데이터 공유한 서브프로세서에 삭제 요청 전달 |
| 폐기 증명 | 삭제 완료 후 삭제 대상/시각/방법 기록 |

### 삭제 불가 항목

| 항목 | 근거 | 보존 기간 |
|------|------|----------|
| 세금계산서/인보이스 | 부가가치세법 | 5년 |
| 결제 거래 기록 | 전자금융거래법 | 5년 |
| 감사 로그 | SOC2 / 정보통신망법 | 5년 |
| 분쟁 관련 데이터 | 민사소송법 | 분쟁 해결 시까지 |

## 데이터 이동권 (Right to Portability)

| 규칙 | 설명 |
|------|------|
| 내보내기 형식 | JSON (기본) + CSV (선택) |
| 범위 | 사용자가 직접 제공한 데이터 + 서비스 내 생성 데이터 |
| 처리 기한 | 요청 후 30일 이내 |
| 대용량 처리 | 비동기 내보내기 → 완료 시 다운로드 링크 이메일 발송 |
| 보안 | 암호화된 ZIP, 일회용 다운로드 링크 (24h 유효) |

## 테넌트 오프보딩

```
해지 요청 → 구독 canceled (유예: 현재 결제 주기 끝)
  → 구독 terminated → 테넌트 suspended (30일 후 테넌트 terminated)
    → 데이터 보존 기간 (90일) → 전체 데이터 삭제
```

| 단계 | 기간 | 접근 | 데이터 |
|------|------|------|--------|
| 유예 (구독 canceled) | 현재 결제 주기 끝까지 | 전체 | 유지 |
| 정지 (테넌트 suspended) | 30일 | 읽기 전용 | 유지 |
| 보존 (테넌트 terminated) | 90일 | 읽기 전용 + 내보내기 | 유지 |
| 삭제 | 보존 기간 종료 | 차단 | 법적 보존 제외 전체 삭제 |

### 오프보딩 필수 조치

| 조치 | 설명 |
|------|------|
| 데이터 내보내기 안내 | 보존 기간 시작 시 내보내기 안내 이메일 |
| API 키 폐기 | 테넌트 suspended 전이 시점에 모든 API 키 즉시 폐기 (구독 canceled 유예 중에는 유효) |
| 웹훅 중단 | 모든 웹훅 발송 중단 |
| 서브프로세서 통지 | 연동된 외부 서비스에 데이터 삭제 요청 |

## 데이터 레지던시

| 규칙 | 설명 |
|------|------|
| 리전 선택 | 테넌트 생성 시 결정, 이후 변경 불가 |
| 저장 제한 | 선택 리전 외 저장 금지 |
| GDPR 역외 이전 | EU 테넌트 → 적정성 결정국 또는 SCC 필수 |
| 로그 포함 | 감사 로그도 동일 리전 저장 |

## 감사 로그

### 필수 기록 이벤트

| 이벤트 | 설명 |
|--------|------|
| 데이터 삭제 요청/실행 | 삭제 대상, 요청자, 실행 시각 |
| 데이터 내보내기 | 내보내기 범위, 요청자, 다운로드 시각 |
| 테넌트 상태 변경 | 변경 전/후, 사유, 실행자 |
| 멤버 추가/제거 | 대상 멤버, 역할, 실행자 |
| 권한 변경 | 변경 전/후 역할, 실행자 |

### 감사 로그 필드

| 필드 | 설명 | 필수 |
|------|------|------|
| timestamp | 이벤트 시각 (UTC) | ✅ |
| tenantId | 대상 테넌트 | ✅ |
| actorId | 실행 사용자 | ✅ |
| actorType | user / system / api_key | ✅ |
| action | 수행 동작 | ✅ |
| resourceType | 대상 리소스 유형 | ✅ |
| resourceId | 대상 리소스 ID | ✅ |
| ipAddress | 요청 IP | ✅ |
| result | success / failure | ✅ |
| detail | 변경 상세 (마스킹) | ⚠️ |

## 참고사항

- 삭제/내보내기 요청은 본인 확인 후에만 처리
- 감사 로그는 불변 (수정/삭제 불가), append-only
- 보존 기간은 프로젝트별 설정 가능 (기본 90일, 최소 30일)
- 대용량 테넌트 삭제는 비동기 배치 처리 (진행률 표시)
