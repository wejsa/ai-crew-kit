# 도메인 로직 체크리스트

SaaS 비즈니스 로직 검증 체크리스트입니다.

## 구독 상태 머신

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 허용 전이만 수행 | subscription-billing.md 허용 전이 테이블 준수 | CRITICAL |
| 상태 변경 이력 | 구독 상태 변경 시 이력 테이블에 전/후/사유/시각 기록 | CRITICAL |
| 테넌트-구독 연동 | 구독 canceled → 유예 종료 → 테넌트 suspended 자동 연동 | CRITICAL |
| 트라이얼 만료 처리 | 결제 정보 미등록 트라이얼 만료 → canceled 전이 | MAJOR |
| 일시정지 기한 | paused 90일 초과 → canceled 자동 전이 | MAJOR |
| 일시정지 재개 | paused → active 전이 시 결제 성공 확인 필수 | MAJOR |

## 과금 계산

| 항목 | 설명 | 심각도 |
|------|------|--------|
| BigDecimal 사용 | 모든 금액 계산 BigDecimal 필수 (float/double 금지) | CRITICAL |
| 세금 계산 | 공급가액 + 부가세(10%) 분리 계산 | CRITICAL |
| 과금 모델 정합성 | 플랜에 정의된 과금 모델(flat/seat/usage/tiered/hybrid)과 실제 계산 일치 | MAJOR |
| 인보이스 항목 분리 | 기본 요금, 시트, 초과, 프로레이션, 크레딧 항목별 분리 | MAJOR |
| 크레딧 처리 | 크레딧 차감 시 음수 인보이스 방지 (최소 0원) | MINOR |

## 프로레이션

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 일할 계산 BigDecimal | (월요금/해당월일수)×잔여일수, HALF_UP | CRITICAL |
| 업그레이드 즉시 적용 | 플랜 업그레이드 시 즉시 차액 청구 | MAJOR |
| 다운그레이드 다음 주기 | 플랜 다운그레이드 시 다음 주기부터 적용 | MAJOR |
| 최소 금액 | 프로레이션 100원 미만 절사 | MINOR |

## 사용량 미터링

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 미터링 정확성 | 사용량 집계와 실제 사용 일치 (최대 5분 지연 허용) | CRITICAL |
| 쿼터 초과 처리 | 하드 리밋 시 HTTP 429, 소프트 리밋 시 경고+허용 | MAJOR |
| 초과 과금 BigDecimal | 초과 사용량 × 단가, BigDecimal HALF_UP | MAJOR |
| 초과 상한 | 초과 요금이 설정된 상한 초과하지 않음 | MAJOR |

## 피처 게이팅

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 플랜별 기능 검증 | API 진입 시 테넌트 플랜과 요청 기능 매칭 확인 | CRITICAL |
| 다운그레이드 시 보존 | 기존 데이터 삭제 없이 읽기 전용 전환 | MAJOR |
| 피처 플래그 캐시 | 캐시 TTL 최대 60초, 변경 후 빠른 반영 | MINOR |

## 테넌트 라이프사이클

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 테넌트 상태 전이 | tenant-isolation.md 허용 전이 테이블 준수 | CRITICAL |
| 프로비저닝 원자성 | 프로비저닝 실패 시 생성 리소스 전체 롤백 | CRITICAL |
| 오프보딩 완전성 | terminated 후 보존 기간 경과 데이터 전체 삭제 확인 | MAJOR |
| suspended 접근 제한 | suspended 테넌트 쓰기/삭제 차단, 읽기만 허용 | MAJOR |

## 웹훅

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 배달 재시도 | 실패 시 exponential backoff 5회 재시도 | MAJOR |
| 서명 검증 | HMAC-SHA256 서명 생성, timestamp 5분 이내 | MAJOR |
| 페이로드 PII 금지 | 웹훅 페이로드에 개인정보 포함 금지 (ID만 전달) | MAJOR |
| 연속 실패 suspended | 10회 연속 실패 시 웹훅 자동 비활성화 | MINOR |

## 성능 / 격리

| 항목 | 설명 | 심각도 |
|------|------|--------|
| Noisy Neighbor 방지 | 단일 테넌트의 과도한 요청이 타 테넌트에 영향 없음 | CRITICAL |
| 테넌트별 Rate Limit | 테넌트/플랜별 API 호출 한도 설정 | MAJOR |
| 쿼리 성능 | tenant_id 포함 인덱스로 full table scan 방지 | MAJOR |
| 커넥션 풀 | 멀티테넌트 환경 커넥션 풀 사이징 (테넌트 수 고려) | MAJOR |
| 배치 작업 격리 | 배치/스케줄러 작업에 tenant_id 컨텍스트 포함 | MAJOR |

## 사용 방법

이 체크리스트는 SaaS 비즈니스 로직 리뷰 시 자동으로 적용됩니다.
테넌트 격리, 과금 정확성, 상태 전이 오류는 CRITICAL로 즉시 수정이 필요합니다.
