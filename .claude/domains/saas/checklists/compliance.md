# 컴플라이언스 체크리스트

SaaS 플랫폼 규정 준수 체크리스트입니다.

## GDPR

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 삭제권 구현 | 사용자 요청 시 30일 이내 개인정보 삭제 | CRITICAL |
| 이동권 구현 | 사용자 요청 시 JSON/CSV 형태로 데이터 내보내기 | CRITICAL |
| DPA 수락 추적 | 테넌트별 데이터 처리 동의(DPA) 수락 이력 기록 | CRITICAL |
| 역외 이전 제한 | EU 테넌트 데이터 적정성 결정국/SCC 없이 역외 이전 금지 | MAJOR |
| 서브프로세서 고지 | 제3자 서비스 사용 시 테넌트에 고지 | MAJOR |
| 쿠키 동의 | 웹 프론트엔드 쿠키 수집 시 사전 동의 | MAJOR |
| 데이터 침해 통지 | 72시간 이내 감독 기관 통지 체계 | MAJOR |

## 개인정보보호법 (PIPA)

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 수집 동의 | 개인정보 수집 시 명시적 동의 | CRITICAL |
| 목적 외 이용 금지 | 수집 목적 외 개인정보 이용/제공 금지 | CRITICAL |
| 최소 수집 | 서비스에 필요한 최소한의 정보만 수집 | MAJOR |
| 보유 기간 명시 | 개인정보 보유 기간 고지 후 기간 경과 시 파기 | MAJOR |
| 제3자 제공 동의 | 개인정보 제3자 제공 시 별도 동의 | CRITICAL |
| 크로스테넌트 격리 | 테넌트 A의 개인정보가 테넌트 B에 절대 노출 불가 | CRITICAL |

## SOC2

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 감사 로그 불변성 | 감사 로그 수정/삭제 불가, append-only | CRITICAL |
| 접근 통제 | 역할 기반 최소 권한 원칙 | CRITICAL |
| 변경 관리 | 코드/인프라 변경 이력 추적 (Git + 리뷰) | MAJOR |
| 인시던트 대응 | 보안 인시던트 발생 시 대응 절차 | MAJOR |
| 암호화 | 전송(TLS 1.2+) + 저장(AES-256) 암호화 | MAJOR |

## 정보통신망법

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 개인정보 처리방침 공개 | 서비스 내 개인정보 처리방침 상시 공개 | CRITICAL |
| 기술적 보호조치 | 개인정보 암호화, 접근 통제, 접속 기록 보관 | MAJOR |
| 접속 기록 보관 | 개인정보 처리 시스템 접속 기록 1년 보관 | MAJOR |

## 데이터 보존/폐기

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 보존 기간 준수 | 결제 기록 5년, 감사 로그 5년, 서비스 데이터 90일 | CRITICAL |
| 자동 폐기 | 보존 기간 경과 데이터 자동 삭제 배치 | MAJOR |
| 폐기 증명 | 삭제 실행 시 대상/시각/방법 기록 | MAJOR |
| 백업 데이터 포함 | 삭제 시 백업 데이터에서도 제거 (다음 백업 주기) | MAJOR |
| 법적 보존 예외 | 세법/소송 근거 데이터는 삭제 대상에서 제외 | MINOR |

## 감사 로그 필수 항목

| 필드 | 설명 | 필수 |
|------|------|------|
| timestamp | 이벤트 시각 (UTC) | ✅ |
| tenantId | 대상 테넌트 | ✅ |
| actorId | 실행 사용자 | ✅ |
| actorType | user / system / api_key | ✅ |
| action | 수행 동작 | ✅ |
| resourceType | 대상 리소스 유형 | ✅ |
| resourceId | 대상 리소스 ID | ✅ |
| ipAddress | 요청 IP | ✅ |
| result | success / failure | ✅ |
| detail | 변경 상세 (민감정보 마스킹) | ⚠️ |

## 사용 방법

이 체크리스트는 SaaS 프로젝트 코드 리뷰 시 자동으로 적용됩니다.
CRITICAL 항목 미준수 시 리뷰 통과 불가합니다.
