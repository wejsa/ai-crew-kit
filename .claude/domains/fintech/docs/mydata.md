# 마이데이터 (본인신용정보관리업)

## 개요

마이데이터 사업자는 정보주체(고객)의 요청에 따라 금융기관 등에 분산된 개인 신용정보를 수집·통합하여 제공합니다. 신용정보법과 마이데이터 가이드라인을 준수해야 합니다.

## 전송요구권 플로우

```
정보주체 → 마이데이터 앱 → 전송요구 → 금융결제원 중계 → 정보제공자(은행 등)
                                                        ↓
                                          정보 전송 → 마이데이터 수신 → 통합 조회
```

### 전송요구 상태 머신

```
requested → consented → transmitting → completed
              ↘ rejected (동의 거부)
                           ↘ failed → retrying → completed
                                                ↘ expired
```

| 상태 | 설명 | 전이 조건 |
|------|------|----------|
| requested | 전송요구 접수 | 정보주체가 앱에서 요청 |
| consented | 동의 완료 | 정보주체 본인인증 + 전송동의 |
| rejected | 동의 거부 | 정보주체 거부 또는 인증 실패 |
| transmitting | 전송 중 | 정보제공자 → 마이데이터 전송 시작 |
| completed | 전송 완료 | 모든 데이터 수신 확인 |
| failed | 전송 실패 | 정보제공자 오류 또는 타임아웃 |
| retrying | 재전송 중 | 자동 재시도 (최대 3회) |
| expired | 만료 | 전송동의 유효기간 초과 |

## 데이터 수집 범위

| 카테고리 | 데이터 항목 |
|----------|-----------|
| 은행 | 계좌 목록, 잔액, 거래내역, 대출 정보 |
| 카드 | 카드 목록, 결제내역, 청구내역, 한도 |
| 보험 | 보험 목록, 보험료, 보장 내역 |
| 투자 | 계좌, 보유 종목, 거래내역 |
| 통신 | 요금 내역, 납부 이력 |

## 동의 관리

| 규칙 | 설명 |
|------|------|
| 개별 동의 | 정보제공자별·항목별 개별 동의 |
| 동의 유효기간 | 최대 1년, 갱신 필요 |
| 동의 철회 | 정보주체 요청 시 즉시 전송 중단 |
| 동의 이력 | 동의/철회 이력 기록 (5년 보존) |
| 재동의 | 수집 범위 변경 시 재동의 필수 |

## 데이터 보관/폐기

| 규칙 | 설명 |
|------|------|
| 보관 기간 | 동의 유효기간 내 보관, 만료 시 즉시 삭제 |
| 분리 보관 | 정보주체별 데이터 격리 |
| 암호화 저장 | 수집 데이터 AES-256 암호화 |
| 폐기 증명 | 삭제 시 폐기 로그 기록 |
| 동의 철회 시 | 5영업일 이내 수집 데이터 전량 삭제 |
| 회원 탈퇴 시 | 즉시 전송동의 철회 + 데이터 삭제 |

## API 규격

### 인증 방식

| 항목 | 설명 |
|------|------|
| 프로토콜 | OAuth 2.0 (Authorization Code Grant) |
| 인증서 | 금융결제원 발급 전자서명 인증서 |
| 토큰 | access_token (유효 기간 정보제공자별 상이) |
| 전송 | HTTPS (TLS 1.2+) 필수 |

### 데이터 전송 주기

| 유형 | 주기 | 설명 |
|------|------|------|
| 정기 전송 | 1일 1회 | 전일 데이터 일괄 전송 |
| 수시 전송 | 실시간 | 정보주체 요청 시 즉시 |
| 갱신 전송 | 동의 갱신 시 | 동의 범위 변경분 반영 |

## 참고사항

- 마이데이터 사업자는 **본인신용정보관리업** 인가 필수 (금융위 등록)
- 수집 데이터를 **마케팅 목적 활용 시 별도 동의** 필요
- 정보제공자 API 장애 시 **사용자에게 지연 안내** 필수
- 정보주체의 데이터 **열람/정정/삭제 요청** 즉시 처리
- 오픈뱅킹 API와의 관계는 `open-banking.md` 참조
