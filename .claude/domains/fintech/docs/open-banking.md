# 오픈뱅킹 플로우

## 개요

금융결제원 오픈뱅킹 API를 통해 타행 계좌 조회, 이체, 출금을 수행하는 플로우입니다. 이용기관(핀테크 사업자)이 금융결제원 중계 서버를 거쳐 은행 API에 접근합니다.

## 이용기관 등록 플로우

```
핀테크 사업자 → 금융결제원 등록 → 이용기관 코드 발급 → API 인증서 발급
                                                     ↓
                                              센터 인증 테스트 → 본인증 통과 → 운영 전환
```

### 필수 준비물

| 항목 | 설명 |
|------|------|
| 이용기관 코드 | 금융결제원 발급 10자리 코드 |
| API 인증서 | 금융결제원 발급 전자서명 인증서 |
| 센터 인증 | 금융결제원 테스트 환경 인증 통과 |
| 이용자 동의 | 계좌 접근 시 고객 사전 동의 필수 |

## 사용자 인증 및 계좌 등록

```
사용자 → 핀테크 앱 → 오픈뱅킹 인증 (OAuth 2.0)
                       ↓
              은행 선택 → 본인 확인 → 계좌 선택 → 출금 동의
                       ↓
              access_token 발급 → 핀테크이용번호(fintech_use_num) 반환
```

### 토큰 관리

| 토큰 | 유효 기간 | 갱신 |
|------|----------|------|
| access_token | 90일 | refresh_token으로 갱신 |
| refresh_token | 1년 | 만료 시 재인증 |

### 토큰 관리 규칙

| 규칙 | 설명 |
|------|------|
| 토큰 암호화 저장 | access_token은 AES-256 암호화하여 DB 저장 |
| 토큰 로깅 금지 | 토큰 평문을 로그에 출력 금지 |
| 갱신 자동화 | 만료 7일 전 자동 갱신 배치 |
| 만료 알림 | refresh_token 만료 30일 전 사용자 알림 |

## 계좌 조회 플로우

```
핀테크 앱 → 오픈뱅킹 API (잔액조회/거래내역조회) → 금융결제원 → 은행
              ↓
        응답 수신 → 데이터 가공 → 사용자 표시
```

### API 엔드포인트

| API | 설명 | 응답 시간 SLA |
|-----|------|-------------|
| /account/balance/fin_num | 잔액 조회 | 3초 |
| /account/transaction_list/fin_num | 거래내역 조회 | 5초 |
| /account/holder/fin_num | 수취인 조회 | 3초 |

### 호출 규칙

| 규칙 | 설명 |
|------|------|
| 거래고유번호 | 모든 요청에 고유번호 부여 (멱등성 보장) |
| 타임아웃 | 기본 3초, 거래내역 5초 |
| 재시도 | 타임아웃 시 최대 1회 재시도 (동일 거래고유번호) |
| 일 호출 한도 | API별 일일 호출 한도 준수 |

## 이체 플로우

```
이체 요청 → 출금이체 API → 금융결제원 → 출금 은행
                                    ↓
                              출금 성공 → 입금이체 API → 입금 은행
                                    ↓
                              입금 성공 → 이체 완료
```

### 이체 상태 머신

```
requested → withdrawn → deposited → completed
              ↘ withdraw_failed → canceled
                         ↘ deposit_failed → refunding → refunded
```

| 상태 | 설명 | 후속 조치 |
|------|------|----------|
| requested | 이체 요청 접수 | 출금이체 API 호출 |
| withdrawn | 출금 완료 | 입금이체 API 호출 |
| deposited | 입금 완료 | 이체 완료 처리 |
| completed | 이체 완료 | — |
| withdraw_failed | 출금 실패 | 사유 코드 확인 → 재시도 또는 취소 |
| deposit_failed | 입금 실패 | 출금 원복(환불) 처리 |
| refunding | 환불 진행 중 | 출금 원복 API 호출 |
| refunded | 환불 완료 | — |

### 이체 필수 검증

| 검증 | 설명 |
|------|------|
| 수취인 확인 | 이체 전 수취인 조회 API로 예금주 확인 |
| 출금 동의 | 사용자 출금 동의 여부 확인 |
| 거래 한도 | 1회/1일/1월 이체 한도 확인 |
| 잔액 확인 | 출금 전 잔액 충분 여부 확인 |

## 에러 처리

| 에러 코드 대역 | 설명 | 대응 |
|---------------|------|------|
| A0000 | 정상 | — |
| A0001~A0099 | 요청 파라미터 오류 | 요청 데이터 검증 |
| A0100~A0199 | 인증/권한 오류 | 토큰 갱신 또는 재인증 |
| A0200~A0299 | 계좌 오류 | 계좌 상태 확인 |
| A0300~A0399 | 한도 초과 | 한도 안내 |
| A0900~A0999 | 시스템 오류 | 재시도 후 장애 리포트 |

## 참고사항

- 오픈뱅킹 API는 **HTTPS(TLS 1.2+)** 전용
- 모든 API 요청/응답은 **JSON** 형식
- 거래고유번호는 **이용기관코드(10) + 생성일(8) + 일련번호(12)** 형식
- 은행 점검 시간(23:30~00:30) 동안 API 이용 불가
- 테스트 환경(센터 테스트)과 운영 환경의 엔드포인트가 다름
- 마이데이터와의 관계는 `mydata.md` 참조
