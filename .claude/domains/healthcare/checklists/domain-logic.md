# 도메인 로직 체크리스트

헬스케어 비즈니스 로직 검증 체크리스트입니다.

## 처방 상태 머신

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 허용 전이만 수행 | prescription-flow.md 허용 전이 테이블 준수 | CRITICAL |
| 처방 수정 금지 | verified 이후 처방 수정 불가 (취소 후 재처방) | CRITICAL |
| 유효기간 만료 처리 | verified 상태에서 유효기간 초과 시 expired 자동 전이 | MAJOR |
| 마약류 유효기간 | 마약류 처방은 3일 유효기간 적용 | MAJOR |

## 약물 상호작용 (DUR)

| 항목 | 설명 | 심각도 |
|------|------|--------|
| DUR 검증 필수 | 처방 저장 시 DUR 자동 검증 실행 | CRITICAL |
| 병용 금기 차단 | CRITICAL 등급 상호작용 발견 시 처방 차단 | CRITICAL |
| Override 시 사유 기록 | 의사가 DUR 경고 무시 시 사유 코드 + 감사 로그 | MAJOR |
| 알레르기 체크 | 환자 등록 알레르기 목록과 처방 약물 대조 | MAJOR |

## 예약 상태 머신

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 허용 전이만 수행 | appointment-flow.md 허용 전이 테이블 준수 | CRITICAL |
| No-show 자동 처리 | 예약 시간 + 대기 시간 초과 시 no_show 자동 전이 | MAJOR |
| 예약 변경 이력 | 예약 변경 시 cancel → 재예약 (수정 금지, 이력 추적) | MAJOR |

## 동의 관리

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 동의 상태 전이 | consent-management.md 허용 전이 테이블 준수 | CRITICAL |
| 동의 없는 접근 차단 | active 상태 동의 없이 해당 목적 PHI 접근 불가 | CRITICAL |
| 철회 즉시 반영 | 동의 철회 시 해당 데이터 접근 즉시 차단 | MAJOR |
| 거부 기록 | 동의 거부(denied)도 반드시 기록 | MAJOR |

## 보험 청구

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 청구 상태 전이 | billing-claims.md 허용 전이 테이블 준수 | MAJOR |
| 수가 코드 유효성 | 폐지/변경된 수가 코드 사용 여부 검증 | MAJOR |
| 중복 청구 방지 | 동일 환자 동일 일자 중복 청구 차단 | MAJOR |

## 진료기록 무결성

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 진료기록 삭제 금지 | 작성된 진료기록 삭제 불가 (추가 기록만 허용) | CRITICAL |
| 수정 이력 | 진료기록 수정 시 원본 보존 + 수정 이력 기록 | MAJOR |
| 타임스탬프 무결성 | 서버 시각 기준 기록, 소급 기록 시 사유 명시 | MAJOR |

## 환자 식별

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 2개 이상 식별자 확인 | 환자 확인 시 최소 2개 식별자 대조 (이름+생년월일 등) | CRITICAL |
| 동명이인 처리 | 동명이인 환자 구분 메커니즘 (추가 식별자 확인) | MAJOR |

## 사용 방법

이 체크리스트는 헬스케어 비즈니스 로직 리뷰 시 자동으로 적용됩니다.
처방 안전성, 동의 검증, 진료기록 무결성은 CRITICAL로 즉시 수정이 필요합니다.
