# PHI 보안 체크리스트

헬스케어 PHI 보호 및 접근 통제 검증 체크리스트입니다.

## PHI 암호화

| 항목 | 설명 | 심각도 |
|------|------|--------|
| PHI 평문 저장 금지 | DB/파일/캐시에 PHI 평문 저장 불가 (AES-256 필수) | CRITICAL |
| 암호화 키 분리 | 암호화 키와 데이터 별도 저장, 연 1회 교체 | CRITICAL |
| 백업 암호화 | 백업 데이터 동일 수준 암호화 적용 | CRITICAL |
| 캐시 PHI 제한 | Redis 등 캐시에 PHI 저장 시 TTL + 암호화 필수 | MAJOR |

## PHI 로깅 금지

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 환자 식별자 로그 금지 | 환자명, 주민번호, 진료번호 로그 출력 절대 금지 | CRITICAL |
| 진단 코드 로그 금지 | ICD-10 코드, 진단명 로그 출력 금지 | CRITICAL |
| 에러 메시지 PHI 제외 | 예외 스택트레이스에 PHI 미포함 설계 | CRITICAL |
| 허용 로깅 | UUID 기반 익명 ID, 이벤트 유형, 타임스탬프만 허용 | MAJOR |

## 접근 통제

| 항목 | 설명 | 심각도 |
|------|------|--------|
| 환자-의료진 관계 검증 | 담당 관계 없는 환자 PHI 접근 차단 | CRITICAL |
| Minimum Necessary | API 응답에 요청 역할/목적에 필요한 최소 PHI만 반환 | CRITICAL |
| 역할별 접근 범위 | 역할-리소스 매트릭스에 따른 접근 제어 (access-control.md) | MAJOR |
| System Admin PHI 차단 | 시스템 관리자 역할에서 PHI 직접 접근 불가 | MAJOR |

## Break-the-Glass

| 항목 | 설명 | 심각도 |
|------|------|--------|
| BTG 사유 코드 필수 | Break-the-Glass 발동 시 응급 사유 코드 입력 필수 | CRITICAL |
| BTG 자동 만료 | 4시간 후 접근 자동 해제 (연장 시 재승인) | MAJOR |
| BTG 즉시 알림 | 발동 시 관리자/감사팀 실시간 알림 | MAJOR |
| BTG 사후 감사 | 모든 BTG 건 사후 정당성 검증 | MAJOR |

## 전송 보안

| 항목 | 설명 | 심각도 |
|------|------|--------|
| PHI 비암호화 전송 금지 | 이메일/팩스 등 비암호화 채널 PHI 전송 금지 | CRITICAL |
| FHIR API 전송 | 외부 연동 시 HL7 FHIR 표준 API 사용 | MAJOR |

## 감사 로그 연동

| 항목 | 설명 | 심각도 |
|------|------|--------|
| PHI 접근 시 감사 로그 | PHI 조회/수정/삭제/출력 시 반드시 감사 로그 기록 | CRITICAL |
| 감사 로그 불변성 | append-only, 수정/삭제 불가 | MAJOR |
| 접근 실패 기록 | 권한 없는 PHI 접근 시도(denied)도 기록 | MAJOR |

## 사용 방법

이 체크리스트는 헬스케어 프로젝트 코드 리뷰 시 자동으로 적용됩니다.
PHI 노출(로깅, 평문 저장, 비암호화 전송)은 CRITICAL로 즉시 수정이 필요합니다.
