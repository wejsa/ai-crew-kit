# 접근 통제

## 개요

PHI 접근을 최소 필요 원칙(Minimum Necessary)과 환자-의료진 관계에 기반하여 제어합니다.

## 역할 계층

| 역할 | 범위 | PHI 접근 수준 |
|------|------|-------------|
| System Admin | 시스템 전체 | PHI 접근 불가 (인프라 관리 전용) |
| 병원 관리자 | 병원/기관 내 | 통계/집계만 (개별 PHI 불가) |
| 의사 | 담당 환자 | 전체 진료기록, 처방, 검사결과 |
| 간호사 | 담당 환자 | 간호기록, 바이탈, 투약 기록 |
| 약사 | 처방 연계 환자 | 처방전, 약물 이력 |
| 접수/원무 | 내원 환자 | 인적사항, 보험정보, 수납 |
| 환자 | 본인 | 본인 진료기록 열람 |

## 접근 제어 매트릭스

| 리소스 | 의사 | 간호사 | 약사 | 접수 | 환자 |
|--------|------|--------|------|------|------|
| 진료기록 | RW | R | — | — | R(본인) |
| 처방 | RW | R | R | — | R(본인) |
| 검사결과 | RW | R | — | — | R(본인) |
| 간호기록 | R | RW | — | — | — |
| 수납/청구 | — | — | — | RW | R(본인) |
| 인적사항 | R | R | R | RW | RW(본인) |

R=읽기, W=쓰기, —=접근 불가

## 환자-의료진 관계 기반 접근

| 규칙 | 설명 |
|------|------|
| 담당 관계 필수 | 의사/간호사는 담당 환자의 PHI만 접근 가능 |
| 관계 설정 | 진료 접수 시 자동 설정, 퇴원/종결 시 해제 |
| 부서 제한 | 타 부서 환자 접근 시 사유 입력 필수 |
| 관계 없는 접근 | 감사 로그 경고 + 관리자 알림 |

## Break-the-Glass (응급 접근)

| 규칙 | 설명 |
|------|------|
| 발동 조건 | 응급 상황에서 담당 관계 없는 환자 PHI 접근 필요 시 |
| 사유 코드 필수 | 응급 사유 선택 (응급실 내원, 의식불명, 아나필락시스 등) |
| 자동 만료 | 4시간 후 자동 접근 해제 (연장 시 재승인) |
| 즉시 알림 | Break-the-Glass 발동 시 관리자/감사팀 실시간 알림 |
| 사후 감사 | 발동 건 전수 사후 감사 (정당성 검증) |
| 감사 로그 | 일반 접근보다 상세한 로그 (사유, 접근 범위, 시간) |

## HL7 FHIR 리소스 매핑

| FHIR 리소스 | 대응 도메인 | 용도 |
|------------|-----------|------|
| Patient | 환자 정보 | 인적사항, 보험, 연락처 |
| Encounter | 내원/입원 | 진료 에피소드 |
| Observation | 검사결과 | 바이탈, 검사값 |
| MedicationRequest | 처방 | 처방 정보 |
| AllergyIntolerance | 알레르기 | 약물/식품 알레르기 |
| Condition | 진단 | 진단 코드 (ICD-10) |
| Claim | 청구 | 보험 청구 |

## 참고사항

- System Admin은 PHI에 접근할 수 없음 (DB 직접 접근도 감사 대상)
- 환자 본인 열람 요청은 30일 이내 제공 (HIPAA Right of Access)
- 제3자 제공 시 환자 동의 필수 (consent-management.md 참조)
- 모든 PHI 접근은 감사 로그 기록 (audit-trail.md 참조)
