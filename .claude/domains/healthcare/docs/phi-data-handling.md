# PHI 데이터 처리

## 개요

PHI(Protected Health Information, 보호건강정보)의 수집, 저장, 전송, 폐기 전 과정에서 HIPAA Privacy Rule과 개인정보보호법(민감정보)을 준수하는 규칙을 정의합니다.

## PHI 18개 식별자 (HIPAA Safe Harbor)

| # | 식별자 | 예시 |
|---|--------|------|
| 1 | 이름 | 환자 성명 |
| 2 | 주소 (시/군 이하) | 상세 주소 |
| 3 | 날짜 (연도 제외) | 생년월일, 입원일, 퇴원일, 사망일 |
| 4 | 전화번호 | |
| 5 | 팩스번호 | |
| 6 | 이메일 | |
| 7 | 주민등록번호 | SSN |
| 8 | 의무기록번호 (MRN) | |
| 9 | 건강보험번호 | |
| 10 | 계좌번호 | |
| 11 | 면허/자격번호 | |
| 12 | 차량번호 | |

## 비식별화 기법

| 기법 | 설명 | 적용 |
|------|------|------|
| Safe Harbor | 18개 식별자 전체 제거 + 재식별 시도 금지 | 연구/통계 데이터 |
| Expert Determination | 통계 전문가가 재식별 위험 "매우 낮음" 판정 | 정밀 분석 필요 시 |
| 가명처리 | 식별자를 가명으로 대체, 매핑 테이블 별도 보관 | 내부 연구 |

## PHI 저장 규칙

| 규칙 | 설명 |
|------|------|
| 암호화 필수 | AES-256으로 저장 시 암호화 (_base/security.md 참조) |
| 키 분리 | 암호화 키와 데이터 별도 저장, 연 1회 키 교체 (HIPAA 164.312) |
| 접근 시 감사 로그 | PHI 조회/수정/삭제 시 반드시 감사 로그 기록 (audit-trail.md 참조) |
| 평문 저장 금지 | DB, 파일, 캐시 어디에도 PHI 평문 저장 불가 |
| 백업 암호화 | 백업 데이터도 동일 수준 암호화 |

## PHI 전송 규칙

| 규칙 | 설명 |
|------|------|
| TLS 1.2+ 필수 | 모든 PHI 전송 시 암호화 채널 (_base/security.md 참조) |
| 이메일/팩스 금지 | 비암호화 채널로 PHI 전송 금지 (환자 명시 동의 시 예외) |
| FHIR API 전송 | 외부 시스템 연동 시 HL7 FHIR 표준 API 사용 권장 |
| API 응답 최소화 | Minimum Necessary — 요청 역할/목적에 필요한 최소 PHI만 반환 |

## PHI 로깅 금지

| 항목 | 규칙 |
|------|------|
| 절대 금지 | 환자명, 주민등록번호, 진료번호, 진단코드를 로그에 출력 금지 |
| 허용 | 익명화된 ID (UUID), 이벤트 유형, 타임스탬프 |
| 마스킹 | 불가피 시 앞 2자 + **** 형식으로 마스킹 후 로깅 |
| 에러 로그 | 스택트레이스에 PHI 포함되지 않도록 예외 메시지 설계 |

## 보존/폐기

| 데이터 유형 | 보존 기간 | 근거 |
|------------|----------|------|
| 진료기록 | 10년 | 의료법 시행규칙 |
| 처방전 | 2년 | 의료법 |
| 수술기록 | 10년 | 의료법 시행규칙 |
| 검사 결과 | 5년 | 의료법 시행규칙 |
| 감사 로그 | 6년 (HIPAA) / 10년 (의료법) | 긴 쪽 적용 |

폐기 시: 복구 불가능한 방식으로 삭제 + 폐기 증명 기록

## 참고사항

- PHI 접근 시 반드시 환자-의료진 관계 확인 (access-control.md 참조)
- 연구 목적 PHI 사용 시 IRB(기관생명윤리위원회) 승인 필요
- 검사 결과(영상, 병리)도 PHI에 해당 — 동일 보호 수준 적용
