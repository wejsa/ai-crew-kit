# 동의 관리

## 개요

환자의 진료, 정보 제공, 연구 참여 등에 대한 동의를 관리합니다. 동의 없는 PHI 처리는 HIPAA 위반입니다.

## 동의 상태 머신

```
requested → granted → active → revoked
  ↘ denied              ↘ expired (유효기간 종료)
```

| 상태 | 설명 | 데이터 접근 |
|------|------|-----------|
| requested | 동의 요청 발송 | 불가 |
| granted | 동의 서명 완료 | 효력 발생 전 (시작일 대기) |
| denied | 환자가 동의 거부 | 불가 (거부 사실 기록) |
| active | 동의 유효 기간 중 | 허용 (동의 범위 내) |
| revoked | 환자가 동의 철회 | 즉시 차단 (법적 보존 제외) |
| expired | 유효기간 종료 | 차단 (재동의 필요) |

### 허용 전이

| From | To | 조건 |
|------|----|------|
| requested | granted | 환자 서명 (전자/수기) |
| requested | denied | 환자 거부 의사 |
| granted | active | 동의 시작일 도래 (즉시 또는 예약) |
| active | revoked | 환자 철회 요청 |
| active | expired | 유효기간 종료 |
| revoked | requested | 재동의 요청 (새 동의서) |
| expired | requested | 재동의 요청 |
| denied | requested | 재요청 (환자 의향 변경) |

## 동의 유형

| 유형 | 설명 | 유효기간 |
|------|------|----------|
| 진료 동의 | 특정 시술/수술에 대한 동의 | 해당 진료 완료까지 |
| 정보 제공 동의 | 타 기관/보험사에 PHI 공유 | 최대 1년, 갱신 필요 |
| 연구 참여 동의 | 임상시험/연구 데이터 사용 | 연구 종료까지 (IRB 승인 범위) |
| 마케팅 동의 | 건강 정보 기반 마케팅 | 최대 1년, 별도 동의 |

## 동의 철회 처리

| 규칙 | 설명 |
|------|------|
| 즉시 효력 | 철회 시점부터 해당 목적의 PHI 접근 차단 |
| 법적 예외 | 이미 수행된 진료 기록은 삭제 불가 (의료법 보존 의무) |
| 연구 예외 | 이미 수집된 익명화 데이터는 철회 대상 아님 |
| 기록 의무 | 동의/철회 이력을 감사 로그에 기록 |

## 응급 예외

| 상황 | 처리 |
|------|------|
| 환자 의식 불명 | 동의 없이 응급 진료 가능 (의료법 제15조) |
| 법정대리인 부재 | 2인 이상 의사 판단으로 진행 |
| 사후 동의 | 환자 의식 회복 후 동의/거부 확인 |
| 기록 의무 | 응급 사유 + 의사 판단 근거 기록 |

## 미성년자 동의

| 규칙 | 설명 |
|------|------|
| 법정대리인 동의 | 만 14세 미만: 법정대리인 동의 필수 |
| 본인 동의 가능 | 만 14세 이상: 일부 항목 본인 동의 가능 |
| 예외 | 성병, 정신건강, 약물 남용 — 법률에 따라 본인 동의만으로 진행 |

## 참고사항

- 동의서 원본은 전자 서명 포함하여 보관 (진료기록 보존 기간 동일)
- 동의 범위 변경 시 기존 동의 revoke + 새 동의 requested
- 동의 거부(denied)도 반드시 기록 — 향후 분쟁 대비
