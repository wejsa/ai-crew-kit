# 감사 추적 (Audit Trail)

## 개요

HIPAA Security Rule과 의료법에 따라 PHI 접근/변경 이력을 불변으로 기록하는 감사 로그 요구사항을 정의합니다.

## 필수 기록 이벤트

| 이벤트 | 설명 | 심각도 |
|--------|------|--------|
| PHI 조회 | 환자 기록 열람 | 필수 |
| PHI 수정 | 진료기록 작성/수정 | 필수 |
| PHI 삭제 | 기록 삭제 (법적 허용 시만) | 필수 |
| PHI 출력 | 진료기록 인쇄/PDF 생성 | 필수 |
| PHI 전송 | 외부 시스템으로 데이터 전송 | 필수 |
| PHI 다운로드 | 환자 데이터 내보내기 | 필수 |
| Break-the-Glass | 응급 접근 발동 | 필수 (상세) |
| 로그인/로그아웃 | 시스템 접근 시작/종료 | 필수 |
| 권한 변경 | 사용자 역할/권한 수정 | 필수 |
| 동의 변경 | 환자 동의 부여/철회 | 필수 |

## 감사 로그 필수 필드

| 필드 | 설명 | 필수 |
|------|------|------|
| timestamp | 이벤트 시각 (UTC, 밀리초) | ✅ |
| userId | 실행 사용자 ID | ✅ |
| userRole | 사용자 역할 (의사/간호사 등) | ✅ |
| patientId | 대상 환자 ID (PHI 접근 시) | ✅ |
| action | 수행 동작 (view/create/update/delete/print/transfer) | ✅ |
| resourceType | 대상 리소스 유형 (진료기록/처방/검사결과) | ✅ |
| resourceId | 대상 리소스 ID | ✅ |
| ipAddress | 요청 IP 주소 | ✅ |
| result | success / failure / denied | ✅ |
| accessJustification | 접근 사유 (비담당 환자 접근 시) | ⚠️ |
| breakTheGlass | BTG 여부 (boolean) | ⚠️ |
| phiFieldsAccessed | 접근한 PHI 필드 목록 | ⚠️ |

## 불변성 요구사항

| 규칙 | 설명 |
|------|------|
| append-only | 감사 로그는 추가만 가능, 수정/삭제 불가 |
| 위변조 방지 | 해시 체인 또는 WORM(Write Once Read Many) 스토리지 |
| 분리 저장 | 비즈니스 DB와 별도 저장소 (관리자도 변조 불가) |
| 타임스탬프 무결성 | NTP 동기화된 서버 시각 사용, 클라이언트 시각 불신 |

## 보존 기간

| 규정 | 기간 | 적용 |
|------|------|------|
| HIPAA Security Rule | 6년 | 미국 대상 서비스 |
| 의료법 시행규칙 | 10년 | 한국 대상 서비스 |
| 적용 규칙 | 두 규정 중 긴 쪽 | 양쪽 대상 시 10년 |

## 열람 기록 제공

| 규칙 | 설명 |
|------|------|
| 환자 요청권 | 환자는 본인 PHI 접근 이력 요청 가능 (HIPAA Right of Access) |
| 제공 기한 | 요청 후 30일 이내 |
| 제공 형식 | 접근 일시, 접근자, 접근 사유, 접근 범위 |
| 예외 | Break-the-Glass 접근도 포함 (비공개 불가) |

## 참고사항

- 감사 로그 저장소는 비즈니스 DB와 물리적으로 분리 권장
- 로그 용량 관리: 오래된 로그는 콜드 스토리지로 이동 (삭제 불가)
- 감사 로그 자체에 PHI를 포함하지 않음 (patientId만 기록, 진단명 등 불포함)
- 접근 시도 실패(denied)도 반드시 기록
